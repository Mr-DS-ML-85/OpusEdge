# Code of Conduct

## The short version

**Be technically direct. Be personally kind. Assume good faith.** That's it.

## The long version

We follow the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

### Expected behaviour

- Focus on what's best for the project and its users.
- Give and gracefully accept constructive feedback. If a review says the math
  is wrong, don't take it personally — check the math.
- Use technical arguments. Cite the paper, cite the code, cite the measurement.

### Unacceptable behaviour

- Personal attacks, dogpiling, or targeted harassment.
- Publishing others' private information without permission.
- Any of the standard bad things ([see the Covenant for the full list](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#our-standards)).

### Enforcement

Reports go to the maintainer via the email on the GitHub profile or via GitHub's
private reporting flow. All reports are handled confidentially. Consequences
scale with severity: a warning → temporary ban → permanent ban.

### Scope

This applies to project spaces (issues, PRs, discussions, related chat) and
to public representation of the project.
